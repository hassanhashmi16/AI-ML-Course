AI Engineer + ML Engineer — Step by Step

One numbered list, top to bottom, zero to complete. Every step: what to learn, one line on why, one link. No lectures.
How to use this: go in order. Don't skip. Each step is small on purpose — when a step says "learn X," go to the linked resource, learn just enough to do the next milestone, then move on. Milestones (gold boxes) are the actual proof you're ready for the next step — don't pass one you can't do. AI Engineering = the front end: building products on top of AI (APIs, RAG, agents). ML Engineering = the back end: understanding and running the models underneath (math, training, infrastructure). This takes you through both, in that order.
PHASE 0 — PYTHON, FROM SCRATCH
1Python syntax basics — variables, printing, comments, indentation-based blocks. docs.python.org
2Control flow — if/elif/else, for, while, list comprehensions. docs.python.org
3Core data structures — list, tuple, dict, set — everything later is built on these. docs.python.org
4Functions — def, default args, *args/**kwargs. docs.python.org
5Classes & OOP — self, __init__, inheritance — every library you'll use is class-based. realpython.com
6Modules & imports — how real projects are organized across files. docs.python.org
7Virtual environments & pip — isolated dependencies per project — like node_modules. docs.python.org
8Files, JSON, try/except — reading data and handling failures without crashing. docs.python.org
9Type hints — sets up Pydantic, which you use starting Step 11. realpython.com
🏁 Milestone: write a script with a function, a class, and file/JSON handling — no syntax lookups.
