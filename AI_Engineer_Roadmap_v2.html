AI Engineer Roadmap

A sequence, not a textbook: what to learn, in what order, why, and what to build with it.
Phase 0 · Python Phase 1 · AI Engineering Phase 2 · ML + MLOps
How to read this: go top to bottom, one stage at a time. Each stage names the libraries to learn (not concepts to memorize), one line on why it matters, the best 1–2 free resources to actually learn it from, and the project it feeds into. Don't move to the next stage until the current project works. Total pace: 5–10 hrs/week, ~62 weeks (~14 months) end to end.
PHASE 0 — PYTHON FUNDAMENTALS Weeks 1–4

Python & Programming Basics

Weeks 1–4

You know JS/React/Next.js well, but Python itself is rusty — and every single library from here on is Python. This closes that gap first, translating from what you already know instead of starting at zero.

Python syntaxfunctions & classesvenv & pipfiles & JSONtype hints
Best resources: Official Python Tutorial (docs.python.org)freeCodeCamp — Python for JavaScript Developers Handbook
Build: A small script — a few variables, one function, one class, reading a JSON file — with no syntax lookups needed.
Checkpoint: you can write basic Python without translating from JS in your head. Not fast — just not foreign.
PHASE 1 — AI ENGINEERING Weeks 5–34 · RAG, agents, evals, deployment

1. Talking to LLMs

Weeks 5–8

This is the actual day-to-day toolkit of AI engineering: calling a model from code, and getting reliable structured data back instead of loose text you have to parse by hand.

Anthropic / OpenAI SDKPydanticasyncio
Best resources: Anthropic's interactive prompt engineering tutorial (free)DeepLearning.AI — ChatGPT Prompt Engineering for Developers (free)
Build (Project 1): A CLI tool that calls an LLM and returns validated, structured (Pydantic) output — not just printed text.

2. RAG & Vector Databases

Weeks 9–14

Lets an LLM answer questions using your own data instead of just what it was trained on — the single most common real-world AI engineering pattern.

Postgres + pgvectorFastAPILangChain text splitters
Best resources: DeepLearning.AI — Retrieval Augmented Generation (free)pgvector official docs
Build (Project 2): A RAG knowledge assistant over a real corpus you actually care about (your own docs, a topic you know).

3. Evals

Weeks 15–18

Proves your system actually works instead of "seemed fine when I tried it a few times" — the single biggest differentiator in the current AI hiring market.

RAGASLangSmith
Best resources: Ragas — Quick Start (free)LangSmith docs
Build: Add a versioned, scored eval suite (30–50 questions) to Project 2 — turn "it works" into a number you can track.

4. Agents

Weeks 19–24

Moves from a fixed "always do A then B then C" pipeline to a system that decides its own next step based on what it finds — think recipe vs. cook.

LangGraphCrewAIMCP
Best resources: LangChain Academy — Intro to LangGraph (free)DeepLearning.AI — Multi AI Agent Systems with CrewAI (free)
Build (Project 3): A multi-step agent with real tool use that can resume after a crash.

5. Agentic RAG (Flagship)

Weeks 25–28

Merges retrieval and agents into one system — the project you'll actually lead with in interviews and freelance pitches. No new libraries; this is combining Stages 2–4.

LangGraph + RAGAS combined
Best resources: reuse Stage 2–4 resources — this stage is integration, not new tools.
Build (Project 4): Merge Projects 2 and 3 into one agentic, fully-evaluated RAG system. This is your flagship.

6. Shipping It

Weeks 29–32

A project that only runs on your laptop doesn't count as production experience — get it live at a real URL.

DockerFly.io / Render / Railway
Best resources: freeCodeCamp — Docker & Kubernetes Fundamentals (free)
Build: Deploy Project 4 to a live URL with basic auth, rate limiting, and logging.

7. Portfolio & Positioning

Weeks 33–34

Makes the work legible to someone who didn't watch you build it. No new tech here — this is writing.

no new tools
Build: A technical case study per project (architecture, decisions, eval results) and a resume rewrite leading with the eval story.
Phase 1 checkpoint: three working, deployed projects — a RAG assistant with evals, a LangGraph agent, and a flagship agentic RAG system — each with a case study you can explain unaided.
PHASE 2 — ML THEORY + MLOPS Weeks 35–62 · The math and infrastructure underneath Phase 1

8. Math & Classical ML

Weeks 35–40

The math underneath everything you built in Phase 1 — now it has real systems to anchor to instead of being abstract theory.

NumPypandasMatplotlibscikit-learn
Best resources: Andrew Ng — Machine Learning Specialization (free to audit)Khan Academy — Linear Algebra (free)
Build (Project 5): A small classical ML project on real data you already own.

9. Deep Learning & Transformers

Weeks 41–48

The actual mechanism (attention) behind every LLM you've been calling since Stage 1 — Stage 8's math, stacked and repeated at scale.

PyTorchHugging Face PEFT
Best resources: Andrej Karpathy — Neural Networks: Zero to Hero (free)Jay Alammar — The Illustrated Transformer (free)
Build (Project 6): A mini transformer built from scratch, or a small model fine-tuned with LoRA.

10. MLOps Core

Weeks 49–54

Turns "works on my machine" into "runs reliably without me watching it."

Docker ComposeKubernetesMLflowGitHub Actions
Best resources: MLOps Zoomcamp — DataTalks.Club (free, 9 weeks)
Build (Project 7): Containerize and deploy a model with a real CI/CD pipeline and model registry.

11. Observability

Weeks 55–60

Know when production is breaking before your users tell you.

PrometheusGrafanaFeastTerraformArgoCD
Best resources: Prometheus official docsMLOps Zoomcamp — Module 5 (monitoring)
Build (Project 8): Full monitoring/alerting added to your flagship agentic RAG system.

12. Integration

Weeks 61–62

One coherent story, not two separate portfolios. No new tools — this is folding Phase 2 back into your Phase 1 flagship.

no new tools
Build: Rewrite the flagship case study to include the MLOps layer, and reposition your resume around the combined skillset.
Done: a production-grade, monitored, agentic RAG system with a full eval suite — backed by real understanding of the math and infrastructure underneath it.
One standing risk: the danger isn't failing to learn AI engineering — it's stopping once Phase 1 "works well enough" and never doing Phase 2. Put a real date on Stage 8 now, before Phase 1 feels finished.
